# `simplicio.install-transaction/v1`

A **planner-produced, pre-mutation** description of what one installer run would do.
Generated by `scripts/install_plan.py:build_plan()` — a pure function of `(runtime,
mode, scope, target, flags)` with no filesystem or network side effects, so it can be
inspected, diffed, and reviewed before anything on disk changes (`--dry-run`).

This is a first, scoped slice of the full contract described in #293 (effect
classification, rollback, smoke tests, N-1→N upgrade proof). Fields present today:

| field | meaning |
|---|---|
| `transaction_id` | Deterministic id derived from `(runtime, mode, scope, target)` — same inputs, same id, so a dry-run plan and the eventual applied transaction can be correlated. |
| `mode` | One of `minimal \| runtime \| full-stack \| ci \| dry-run \| rollback` (#293 § Modos). `minimal` is the only mode implemented as a real installer path today; the others are modeled in the schema for the planner to grow into. |
| `scope` | `project \| user \| system` — where the mutation lands. |
| `files` | Every file the installer would create or update, each tagged `reversible: true/false` and an `action` (`create`/`update`/`skip` when already present). |
| `permissions_required` | Explicit list of consent-requiring effects this plan would trigger (`global_package`, `break_system_packages`, `path_write`, `symlink`, `service`, `proxy`, `browser`, `secrets`) — empty for a `minimal` project install, non-empty the moment `--global`, a service, or a proxy is requested. |
| `status` | `PLANNED` for every plan `build_plan()` returns; `APPLIED`/`ROLLED_BACK`/`BLOCKED` are reserved for the (not yet implemented) executor/rollback phases. |

## Non-goals of this slice

- No executor consumes this plan yet — `scripts/install_lib.py --dry-run` prints/writes
  it and exits without mutating, but the normal (non-dry-run) install path is
  unchanged and does not yet execute *from* the plan.
- No rollback, no backup/restore, no hash-based idempotency check against a prior
  transaction receipt. Those remain open per #293's own step-by-step plan.
