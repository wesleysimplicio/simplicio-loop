## Reproducing this report

```
python3 scripts/repo_history_scan.py --write-report
```

This regenerates both this file and `docs/repo_size_report.json` from the CURRENT git object database of the clone it runs in — the numbers will differ slightly between clones only if the underlying refs/objects differ (e.g. a shallow clone). Run it from a full, unshallowed clone for numbers that match the canonical remote.
